# FLUX-ISA Constraint-Theory Virtual Machine Instruction Set Specification
**Document ID**: FLUX-ISA-OPCODES-v1.0  
**Domain**: Constraint-theory optimized virtual machine instruction set

---

## 1. Core Enumeration Specification
The FLUX-ISA uses a single-byte (`repr(u8)`) typed opcode enumeration named `FluxOpcode`, with standard traits for debuggability and interoperability:
```rust
#[derive(Debug, Clone, Copy, PartialEq, Eq, Hash)]
#[repr(u8)]
pub enum FluxOpcode {
    // All opcodes grouped by functional category per official allocation
}
```
All opcodes are assigned to non-overlapping contiguous hexadecimal ranges, with explicit mnemonics defined per the spec.

## 2. Primary Functional Opcode Categories
### 2.1 ARITHMETIC (0x01 â€“ 0x05)
Basic arithmetic operations, allocated hex range 0x01 to 0x05:
| Hex Opcode | Mnemonic | Operation Description |
|---|---|---|

### 2.2 CONSTRAINT (0x10 â€“ 0x13)
Constraint validation and enforcement operations, allocated hex range 0x10 to 0x13:
| Hex Opcode | Mnemonic | Operation Description |
|---|---|---|
| 0x10 | `Assert` | Post constraint condition for enforcement |
| 0x11 | `Check` | Validate pending constraint conditions |
| 0x12 | `Validate` | Verify full constraint set satisfaction |
| 0x13 | `Reject` | Reject violated constraints or constraint sets |

### 2.3 FLOW (0x20 â€“ 0x24)
Execution flow control operations, allocated hex range 0x20 to 0x24:
| Hex Opcode | Mnemonic | Operation Description |
|---|---|---|
| 0x20 | `Jump` | Unconditional branch to target instruction offset |
| 0x21 | `Branch` | Conditional branch to target offset based on status flags |
| 0x22 | `Call` | Invoke subroutine with return address storage |
| 0x23 | `Return` | Exit current subroutine and resume execution |
| 0x24 | `Halt` | Terminate virtual machine execution immediately |

| Hex Opcode | Mnemonic | Operation Description |
|---|---|---|

| 0x33 | `Pop` | Pop top stack value into a general-purpose register |
