# Lighthouse Runtime Technical Specification

### System Domain: PLATO Agent Room Model Orchestration Platform

## 1. System Overview
Lighthouse Runtime is a purpose-built model orchestration framework for the PLATO agent ecosystem, designed to automate end-to-end task execution via standardized agent room management, cost-optimized model selection, secure output validation, and isolated per-agent state tracking.

## 2. Core Execution Pipeline
The platform follows a linear, 3-stage pipeline for all task workflows, with each stage defined explicitly:
### 2.1 Orient Stage (`orient(task)`)
This initial pipeline stage performs three critical functions:
1.  Maps the submitted task to eligible models per the predefined Task-to-Model Assignment rules
2.  Selects the least expensive eligible model from the matching eligible pool
3.  Provisions a dedicated, isolated agent room for the task’s full execution lifecycle
### 2.2 Relay Stage (`relay(room, seeds)`)
Following successful agent room provisioning:
1.  Injects provided seed data into the agent room’s dedicated filesystem directory
2.  Launches the agent instance tied to the cost-optimized selected model, which begins task execution once seeding is finalized
### 2.3 Gate Stage (`gate(output)`)
Post-execution validation ensures only safe, verified output is released downstream:
1.  Scans completed task output for three prohibited risk categories: unauthorized credential leaks, unsubstantiated overclaims, and triggered external actions
2.  Rejects any output that fails validation and marks the associated agent room as `failed`; routes valid output to downstream consumers

## 3. Per-Agent Filesystem Hierarchy
Each agent room has an isolated, standardized filesystem structure rooted at `state/agents/{room_id}/`, where `{room_id}` is a unique, system-generated identifier assigned during the Orient stage. The hierarchy includes the following mandatory files and directories:
| Path Component | Description |
|----------------|-------------|
| `state.json` | Persisted runtime state file tracking task progress, model interaction history, and execution metadata for the agent room |
| `tiles/` | Directory for modular task tiles, execution templates, and reusable task subunits |
| `bottles/` | Directory for cached interim model outputs, packaged asset blobs, and partial generation results |
| `log/` | Directory for structured and unstructured runtime logs generated by the agent, model API calls, and pipeline stages |
| `seeds/` | Directory holding initialization seed data injected during the Relay stage, including reference prompts, datasets, and pre-configured execution parameters |

## 7. Agent Lifecycle Status Enumeration
The Lighthouse Runtime uses a standardized enum to track the current state of all agent rooms and associated agents. Valid enum values are:
| Status Value | Definition |
|--------------|------------|
| `orienting` | Agent room is being provisioned, and task-to-model selection is in progress |
| `seeding` | Seed data is being injected into the agent room prior to agent launch |
| `running` | Agent is actively executing the assigned task |
| `paused` | Task execution has been temporarily suspended |
| `complete` | Task execution finished successfully and output passed all gate validation checks |
| `failed` | Task execution encountered a fatal error or failed gate validation checks |
