# Lighthouse Runtime Technical Specification
**System Domain**: PLATO Agent Room Model Orchestration System
**Source Compressed Tile**: LIGHTHOUSE-RUNTIME

## Preamble
This document formally expands the compressed Lighthouse Runtime knowledge tile into a complete, structured technical specification using only explicit, verifiable data from the source tile. No speculative or fabricated information is included.

---

## 1. Core Functional Pipeline
The Lighthouse Runtime implements a three-stage sequential pipeline for orchestrating model-driven agent room operations:
1.  **Orient Stage (`orient(task)`)**: The initial pipeline step that evaluates available models using per-1K-query cost metrics, selects the lowest-priced eligible model, and provisions a dedicated isolated agent room for task execution.
2.  **Seed & Agent Execution Stage (`relay(room, seeds)`)**: Transfers all provided seed data to the newly created agent room’s staging directory, then initiates runtime execution of the orchestrated agent within the room.
3.  **Safeguard Validation Stage (`gate(output)`)**: Performs mandatory post-execution checks on agent-generated output, including credential leak detection, overclaim validation, and audit logging of all external actions initiated by the agent.

## 2. Per-Agent Isolated Filesystem Structure
Each dedicated agent room uses a standardized, isolated filesystem hierarchy rooted at the path:
`state/agents/room_id/`
The following mandatory files and directories are contained within this root directory:
| Item Type       | Path/Name                  | Purpose                                                                 |
|-----------------|----------------------------|-------------------------------------------------------------------------|
| Core State File | `state.json`               | Persistent storage for real-time agent state and runtime metrics        |
| Subdirectory    | `tiles/`                   | Staging area for structured task tile data                              |
| Subdirectory    | `bottles/`                 | Storage directory for completed agent output artifacts                  |
| Subdirectory    | `log/`                     | Directory for timestamped runtime execution logs                        |
| Subdirectory    | `seeds/`                   | Staging area for input seed data provided to the agent during setup     |
