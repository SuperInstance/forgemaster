### The Right-Skew Design Principle (18.5KB Paper)
- **File:** `research/RIGHT-SKEW-DESIGN-PRINCIPLE.md`
- Deep research subagent (Seed-2.0-pro via Claude Code wrapper)
- Proven: CDF theorem for arbitrary lattices in arbitrary dimensions
- Known in lattice quantization theory (Conway & Sloane, Gray & Neuhoff) but engineering implications not discussed
- **Key theorem:** PDF grows as r^(n-1). In 2D, f(r) = 2πr/A (linear). In 8D, f(r) ∝ r⁷ (extreme skew).
- **Square-root funnel** δ(t) = √(A·(1-t)/π) replaces exponential — matches error geometry
- **Sensor placement:** Optimal positions at CDF quantiles, density ∝ 1/r (more sensors near boundaries)
- **INT8 with square-root companding:** code k → radius √(k/256)·ρ, fine at boundaries, coarse at center
- **Design principle:** "Design for the Boundary" — boundary IS the common case, center is exception
- **Anti-patterns:** symmetric deadbands, uniform sensor grids, exponential convergence expectations
- **Non-lattice extension:** holds for ALL Voronoi tessellations (Poisson, centroidal, etc.)
- **Non-uniform inputs:** Gaussian input → Rayleigh CDF, ALSO right-skewed
- **Higher dimensions:** median approaches ρ as n → ∞ — virtually every point near boundary
