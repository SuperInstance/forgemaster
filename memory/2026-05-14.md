### AVX-512 Experiment: The Fast-Math Finding (2026-05-14 ~00:30 AKDT)

**Sandbox experiment on eileen (Ryzen AI 9 HX 370, 24 cores)**

Tested Eisenstein snap across vector widths and memory layouts. The findings:

| Config | ns/snap | throughput | insight |
|--------|---------|------------|---------|
| Scalar AoS | 14.4 | 69M/s | baseline |
| AVX-512 AoS | 14.3 | 70M/s | NO improvement |
| AVX-512 SoA | 26.2 | 38M/s | SLOWER (strided without vectorization) |
| **AVX-512 SoA + fast-math** | **1.6** | **621M/s** | **9× speedup!** |
| @ 10M points | 1.7 | 597M/s | holds at scale |

**Root cause**: The bottleneck isn't vector width — it's `round()` having strict IEEE
semantics. Without `-ffast-math`, GCC can't reorder/associate the rounding and distance
computations. With it, the SoA layout vectorizes across 8 doubles per AVX-512 register.

**Accuracy verified**: max snap distance 0.540 < bound 0.577, 0 idempotence failures.
`-ffast-math` does NOT break correctness for this workload because the snap result is
still rounded to integers and the 3×3 neighbor search picks the true nearest.

**This is what sandboxes are for**: Isolated experiments, clean teardown, PLATO results.

### Key Design Decisions

1. **Pinna as fixed geometry**: No learning needed — the schema IS the encoding. Agents develop "spatial hearing" through calibration history.
2. **Disproof-only admission**: After seed phase (first 50 tiles), new tiles must falsify existing ones. Loop/spline tiles exempt. From MULTI-MODEL-SYNTHESIS.md Novel Idea 2.
3. **Mortality sweep**: Bottom 15% by win/loss ratio pruned each cycle. Never by embedding similarity. Protects loop/spline/meta tiles.
4. **Contamination is continuous**: Not binary frame-intact/collapsed. The sensor measures degradation level (CLEAN/MILD/MODERATE/SEVERE) and trend (WORSENING/IMPROVING/STABLE).
5. **Jam mode for identical PARTIAL agents**: Division of labor (A computes pieces at T=0.0, B combines at T=0.3) instead of iteration (which hurts due to anchoring bias).
6. **Graduation requires all 3 markers**: tiles_retrieved_by_others ≥ 3, capability_card_registered, done_outputs_consumed_as_data ≥ 1. Agent never sees these.
7. **Conservation law checker**: Tests seed-pro's prediction that echo + partial + correct stays flat (87-93%) across the ECHO→PARTIAL transition. If confirmed → first-order phase transition. The single most important falsifiable test.

### Critical Bug Found and Fixed: Reasoning Content Extraction
Multiple models (MiMo-V2.5, Qwen3.5-4B, and partially Qwen3.5-0.8B) put their actual answers in `reasoning_content` field, not `content`. The `content` field is empty or contains only echo/partial text. Same issue we discovered with GLM-5.1 on z.ai earlier.

**Fix pattern:**
```python
# Check content first, fall back to reasoning_content
content = choice.get("message", {}).get("content", "")
reasoning = choice.get("message", {}).get("reasoning_content", "")
text = content if content.strip() else reasoning
```

This is now a FIRST-CLASS VARIABLE — extraction method determines whether a model measures 0% or 86%.
