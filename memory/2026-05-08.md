## Philosophical Breakthrough: The Feedback Loop IS Intelligence

Four research docs written from conversation with Casey:

1. **FEEDBACK-LOOP-INTELLIGENCE.md** — Expertise is in the feedback loop, not the model. Musician twists knobs, hears delta, converges. Painter mixes pigments, sees delta, converges. Grandmother tastes, adds, tastes again. The loop IS the intelligence.

2. **VALUES-AS-ATTRACTORS.md** — Constraints ARE values. The delta (change between states) carries more information than the state. Values exist as attractors in the state space whether or not words describe them. Words come after, as lossy compression.

3. **SET-AND-SETTING.md** — Same action, different context = different intelligence. Physics-clock reads the setting. Constraint margins read the set. The agent adapts its convergence strategy based on context — tight oscillations under stress, wide exploration under calm. Like a musician in a cathedral vs a basement.

4. **THINKING-VS-WEIGHING.md** — LLMs weigh (one forward pass, statistical output). The musician thinks (oscillate through deltas toward an attractor). Lower in the stack = closer to thinking because the physics is exposed. Gate delays, thermal coupling, constraint margins — these are perceptual channels, not metrics.

5. **FULL-STACK-PLAN.md** — Build one unbroken circuit from physics to meaning. Not 20 separate repos talking via JSON. One binary, one loop: read physics → evaluate constraints → perceive delta → compare to attractor → perturb → repeat. Hot loop (performance, no allocation) + cold loop (practice, exploration). Three phases: fleet-agent-core (Rust), fleet-agent-fpga (Verilog), fleet-consciousness (multi-agent).
