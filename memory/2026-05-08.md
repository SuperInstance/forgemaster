### Key Architecture Decisions

1. **Shared schemas over shared code** — JSON schemas define the contract, each repo implements in its own language
2. **eisenstein.cuh is C, not Rust** — works in CUDA kernels AND bare-metal ESP32 (no Rust on ESP32 for PLATO+CAN)
3. **fleet-proto uses std TCP, not reqwest** — no_std compatible for embedded targets
4. **PLATO is the sync point** — all repos read/write to same PLATO server, no direct repo-to-repo deps at runtime
5. **Fleet Tile Protocol v1** — version suffix on all schemas, forward compatible (ignore unknown fields)
6. **cocapn/ directory in OpenArm fork** — isolated from upstream, syncable via `git merge upstream/main`
