# 2026-04-30 Session Log

## Root Cause Found & Fixed: deepseek-v4-flash 400 Error

### The Problem
OpenClaw was failing with: `"400 The reasoning_content in the thinking mode must be passed back to the API."`

### Root Cause Investigation
1. **Config issue #1:** `deepseek-v4-flash` model was NOT registered in `openclaw.json` models list
2. **Config issue #2:** Workspace credential file had stale API key (active key is in `auth-profiles.json`)
3. **Actual root cause:** `deepseek-v4-flash` had `"reasoning": true` in model definition, causing OpenClaw's `buildOpenAICompletionsParams` to send `reasoning_effort: "high"` (default from `resolveOpenAICompletionsReasoningEffort`)
4. DeepSeek's `reasoning_effort` param triggers thinking mode, which returns `reasoning_content` in response
5. On follow-up tool call turns, the `reasoning_content` MUST be passed back, but OpenClaw's message transformation was not properly round-tripping it

### The Fix
Set `"reasoning": false` for `deepseek-v4-flash` in model definition.
- When `reasoning: false`: `resolveThinkingDefaultForModel()` returns "off" → no `reasoning_effort` param sent → DeepSeek doesn't enter thinking mode → clean tool calls without `reasoning_content`

### Verification
- Reproduced the exact 400 error via curl by sending `reasoning_effort: "high"` without round-tripping `reasoning_content`
- Verified the fix works: no `reasoning_effort` → no `reasoning_content` → no round-trip issue
- Verified `reasoning_effort: "high"` DOES trigger thinking (even "low" does)
- Verified NOT sending `reasoning_effort` at all avoids thinking mode entirely

### Config Change
- File: `~/.openclaw/openclaw.json`
- Changed `deepseek-v4-flash.reasoning` from `true` → `false`
- Also updated stale credential file at `~/.openclaw/workspace/.credentials/deepseek-api-key.txt`
- Registered `deepseek-v4-flash` model definition (was missing entirely)
