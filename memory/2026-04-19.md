# Memory — 2026-04-19 (Session N+6+)

## Sprint: JC1 Research Integration + 3 New Crates

### 8 Crates Pushed (130 new tests)

**Upgraded (JC1 Living Knowledge + Merge/Split integration):**
- `plato-tile-dedup` v2 — 18 tests. 4-stage similarity: exact→keyword(Jaccard)→embedding(cosine)→structure(QuestionType), weighted 0.1/0.3/0.5/0.1. Merge strategies (absorb/enrich/synthesis/conflict). Split heuristics (size/complexity/topic_drift). Bag-of-words for keyword stage. thresholds: merge 0.85, split_size 500, split_complexity 5 topics.
- `plato-tile-spec` v2.1 — 24 tests. Added: usage_count (u64), success_rate (f64), provenance (TileProvenance: created_by, validation_method, source_room), dependencies (Vec<String>). TileProvenance::default() = "unknown"/"none"/"". from_living_knowledge() convenience constructor. is_well_established() = usage_count >= 10 && success_rate >= 0.8.
- `plato-tile-store` v2 — 17 tests. Immutable tiles: insert returns version 1, insert_version creates new versions with parent_id. get() = latest, get_version() = specific. restore() rolls back to specific version (creates new child). cascade_invalidate() propagates to dependents. version_history() returns all versions. All version IDs are u32.
- `plato-tile-scorer` v2 — 16 tests. Controversy signal: challenged tiles (challenge_count > 0) get counterpoint_survival_bonus = min(challenge_count * 0.05, 0.3). Sunset clause: unchallenged tiles decay via days_since_creation * 0.01 (capped at 0.3 penalty). Controversy weight = 0.10 in 6-signal scoring. Overall weights: temporal 0.15, ghost 0.15, belief 0.25, domain 0.20, frequency 0.10, keyword 0.30, controversy 0.10.
- `plato-forge-listener` v2 — 20 tests. Gap detection: if response confidence < 0.7 → create GapTile with priority formula (frequency * urgency * impact). Priority thresholds: P0 >= 5.0, P1 >= 2.0, P2 < 2.0. fulfill_gap() marks gap as fulfilled. GapTile stores: gap_id, description, context, frequency, urgency, impact, priority, fulfilled_at.

**New crates:**
- `plato-tile-graph` — 14 tests. Directed dependency DAG. BFS impact_radius(), downstream()/upstream() traversal, roots()/leaves(), cycle detection (DFS), topological_sort() (Kahn's), critical_path_length(). remove_tile() returns orphaned dependents.
- `plato-room-persist` — 10 tests. JSONL event journal: Enter/Leave/Message/StateChange/TileAdd/TileRemove/Snapshot. to_jsonl()/from_jsonl() roundtrip. RoomJournal: events_for_room(), events_since_snapshot(), get_state() (latest wins), agents_in_room() (enter/leave tracking), tiles_in_room() (add/remove tracking), compact() (discard pre-snapshot events).
- `plato-tile-validate` — 11 tests. 6 validation gates: confidence (min 0.3), content_length (10-100k), freshness (7-day window), usage_quality (new tiles exempt), domain_format (non-empty), similarity (Jaccard, threshold 0.9). acceptance_threshold = 0.6. TileInput struct with all v2.1 spec fields.

### Key Decisions This Sprint
- **Bag-of-words for keyword stage in dedup**: `split_whitespace().collect::<HashSet<&str>>` — no stemming, fast, good enough for keyword matching.
- **u32 for version IDs in store**: matches plato-tile-spec versioning, sufficient range.
- **events_since_snapshot includes snapshot event**: user can filter if needed, simpler API.
- **Controversy signal separate from existing 6 signals**: added as 7th with weight 0.10, keyword weight stays at 0.30.
- **Sunset clause uses days not hours**: `created_at / 86400` — longer timescale appropriate for knowledge tiles.
- **New tiles (usage_count=0) always pass usage_quality check**: encourages contribution without prior validation.
- **extract_field uses "field":"pattern"**: simple JSON parser, works for string fields but ts is numeric — used separate numeric extraction.
