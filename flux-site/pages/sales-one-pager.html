FLUX — The First Certifiable Constraint Compiler

The Problem

Aerospace control logic verification remains the largest schedule and cost risk on DO-254 / DO-178C programs. Current commercial toolchains require $2–5M in licensing and qualification labor per program, with formal certification artifacts taking 18–36 months to produce. No existing tool provides end-to-end correctness guarantees between requirement specification and deployed machine code. Manual proof work introduces human error at every stage.

The Solution

FLUX is a constraint compiler that translates formal system requirement specifications directly to runtime executable machine code, with mathematical proofs of correctness generated automatically for every compilation pass. No intermediate manual verification is required. All output artifacts are acceptable for direct submission to certification authorities without additional tool qualification work.

How It Works

  1. Engineers write constraint specifications in GUARD, a human-readable domain specific language for aerospace requirements
  2. FLUX compiles GUARD to formally verified FLUX-C bytecode
  3. Bytecode is translated to native target code, with formal equivalence proofs generated for every transformation

Why FLUX

FLUX executes on a purpose-built Turing-incomplete virtual machine. This guarantees decidable halting behavior for all valid input, statically computable worst case execution time (WCET), fully deterministic repeatable builds, and elimination of all undefined runtime behavior. These properties are required for highest design assurance level certification, and cannot be achieved with general purpose compilers.
