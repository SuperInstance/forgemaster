See It In Action

GUARD
// Altitude constraint: aircraft must stay
// within operational ceiling during flight
constraint altitude_safety {
    let max_alt = 45000   // feet MSL
    let min_alt = 500     // AGL floor
    let rate_limit = 6000 // ft/min

    require alt in [min_alt, max_alt]
    require |d_alt/dt| <= rate_limit
    require vertical_sep >= 1000

    on violation {
        emit "ALT_RANGE"
        trigger autopilot_override
    }
}
FLUX-C Bytecode

Safety Case — GSN Summary

Goal Structuring Notation (GSN) breaks the top-level safety claim into 7 independently verifiable sub-goals.

G1 — Compiler Correctness
guard2mask preserves constraint semantics
Proved in Coq: compilation is a semantics-preserving transformation from GUARD IR to verification masks.
G2 — Kernel Correctness
FLUX-C generates correct GPU/FPGA kernels
Each kernel template is verified against its mask specification. No undefined behavior reachable.
G3 — Runtime Integrity
Execution environment is sound
Memory safety, no data races, deterministic reduction. Proved via separation logic.
G4 — No False Negatives
Every violation is detected
Soundness proof: if a constraint is violated, FLUX will emit a violation signal. No gaps.
G5 — No False Positives
Every alert is real
Completeness proof: if FLUX emits a violation, the constraint was truly violated. No noise.
G6 — Timing Guarantees
Worst-case execution time bounded
WCET analysis provides hard bounds for each platform. Suitable for hard real-time systems.
G7 — Certification Readiness
DO-178C/ISO 26262 artifacts traceable
Full traceability from requirements → GUARD → proofs → code. Generated evidence packages.
