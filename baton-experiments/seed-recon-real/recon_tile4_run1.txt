# Technical Specification: Eisenstein-Dodecet 12-Bit Constraint Encoding Module
*Derived verbatim from compressed knowledge tile [EISENSTEIN-DODECET]; no fabricated or external data added*

---

## 1. Core Domain & Purpose
This module defines a standardized 12-bit state encoding (dodecet) framework for constraint management on the A2 Eisenstein integer lattice. It is purpose-built to compactly encode snapping deviation, orientation, safety, and lattice symmetry metadata for 2D lattice projection and snapping operations.

---

## 2. Dodecet Bit-Layout Specification
The full 12-bit dodecet state is partitioned into three 4-bit nibbles, with explicit fixed bit assignments for each functional field:
### 2.1 Nibble 2 (Bits 11–8: Most Significant Nibble)
- Value range: 0 (0x0) to 15 (0xF)
- Functional role: Encodes constraint-level deviation from an ideal lattice snap point. A value of `0` corresponds to an on-snap state (zero positional deviation), while a value of `15` corresponds to exactly reaching the lattice's covering radius ρ.
### 2.2 Nibble 1 (Bits 7–4: Middle Nibble)
- Value range: 0 (0x0) to 15 (0xF)
- Functional role: Encodes direction-in-cell azimuth angle. Each integer value maps to a 22.5° azimuth increment, delivering 16 evenly spaced angles spanning 0° to 337.5° (full 360° coverage).
### 2.3 Nibble 0 (Bits 3–0: Least Significant Nibble)
Split into two sub-fields for chirality and safety metadata:
  1.  **Chamber Identifier (Bits 0–2):** 3-bit unsigned integer with valid range `0–5`, corresponding to one of 6 Weyl symmetry chambers of the A2 lattice Voronoi cell. Values 6–7 are reserved per spec.
  2.  **Safety/Critical Flag (Bit 3):** Single boolean flag indicating safety state. A reference critical calibration threshold of `0.70` is explicitly provided.
This nibble encodes both chirality and safety metrics for the snapped state.

## 3. Mathematical Foundation Constants
All lattice and symmetry constants follow canonical A2 Eisenstein integer lattice specifications:
- **A2 Lattice:** The 2-dimensional hexagonal integer lattice, equivalent to the ring of Eisenstein integers ℤ[ω].
- **Covering Radius (ρ):** Maximum Euclidean distance from any point within a Voronoi cell to the nearest lattice point. Exact value: $ρ = 1/\sqrt{3} ≈ 0.5774$.
- **Voronoi Cell Area:** Fundamental domain area per A2 lattice point. Exact value: $\sqrt{3}/2 ≈ 0.8660$.
- **Primitive 3rd Root of Unity (ω):** Euler's formula expanded to rectangular form: $ω = e^{2πi/3} = -1/2 + i·\sqrt{3}/2$.
- **Weyl Group (S₃):** Finite symmetry group of the equilateral triangle, with order 6 (6 total symmetry operations: 3 rotations, 3 reflections).

---

## 4. Weyl Chambers & Parity Classification
The A2 lattice's Voronoi cell is divided into 6 congruent triangular chambers, each tied to a permutation of the standard lattice basis coordinate set `(0,1,2)`. Chambers are categorized by symmetry parity:
1.  **Even-Parity Chambers:** Reachable via rotational lattice symmetries. Valid chamber indices: `[0, 2, 5]`.
2.  **Odd-Parity Chambers:** Reachable via reflection lattice symmetries. Valid chamber indices: `[1, 3, 4]`.

This parity classification directly maps to the `parity` field of the `SnapResult` struct.

## 5. Safety & Constraint Thresholds
1.  **Global Safe Threshold:** Defined as half the lattice covering radius: $ρ/2 ≈ 0.2887$. Input points with deviation distance below this threshold are classified as safe; points at or above this threshold are critical.
2.  **Constraint Level Distribution:** 70% of all valid constraint level values (0–15) fall at or above level `8`, meaning the majority of operational constraint states lie in the upper half of the deviation scale.
3.  **Critical Reference Calibration:** A fixed reference value of `0.70` is provided to align safety flag configuration with industry-standard critical deviation thresholds.

## 6. SnapResult Data Structure
The primary output struct returned by all snapping operations, containing all encoded state and computed metrics:
| Field Name               | Type       | Exact Specification                                                                 |
|--------------------------|------------|-------------------------------------------------------------------------------------|
| `dodecet`                | `u16`      | Compact 12-bit encoded state, padded to a 16-bit unsigned integer for standard alignment |
| `snap_a`, `snap_b`       | `i32`      | 32-bit signed integer coordinates of the two closest Eisenstein lattice snap points, formatted as Eisenstein integer coefficients $a + b·ω$ |
| `error`                  | `f64`      | Euclidean distance between the input point and the nearest snap lattice point       |
| `error_normalized`       | `f64`      | Normalized deviation scaled to the covering radius ρ, bounded strictly to $[0, 1]$ |
| `error_level`            | `u8`       | 4-bit constraint level, matches Nibble 2 of the dodecet state (range 0–15)           |
| `angle_level`            | `u8`       | 4-bit azimuth direction value, matches Nibble 1 of the dodecet state (range 0–15)    |
| `chamber`                | `u8`       | Weyl chamber index (range 0–5), matches the chamber subfield of Nibble 0             |
| `parity`                 | `i8`       | Symmetry parity, either `+1` (even) or `-1` (odd)                                   |
| `is_safe`                | `bool`     | Safety classification flag: `true` if deviation is below $ρ/2$, `false` otherwise     |

## 7. Hexadecimal Encoding & Public API
1.  **3-Character Hex Format:** The 12-bit dodecet state is serialized to a compact 3-character hexadecimal string, where each character maps directly to one of the three 4-bit nibbles in order (Nibble 2 → Nibble 1 → Nibble 0).
2.  **Public Programming Interface:** The module’s core entry point is the static method `EisensteinConstraint::new().snap(x, y)`, which accepts two floating-point Cartesian coordinates `x` and `y` as input, executes the snapping constraint calculation, and returns a fully populated `SnapResult` struct.
