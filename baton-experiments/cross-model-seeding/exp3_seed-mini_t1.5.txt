## 2. $\boldsymbol{\text{snap}(x,y)}$ Nearest-Lattice Rounding Function
### 2.1 Formal Algorithm & Constant-Time Guarantees
The `snap` function maps arbitrary real-valued Cartesian coordinates $(x,y) \in \mathbb{R}^2$ to the unique nearest $E_6$ lattice point, with **O(1)** execution time per call:
1.  Estimate integer axial coordinate $r$ by solving for the nearest match to the Y-coordinate: $\hat{r} = \text{round}\left(\frac{2Y}{\sqrt{3}}\right)$
2.  Estimate integer offset coordinate $q$ using the current estimate of $r$: $\hat{q} = \text{round}\left(X - \frac{\hat{r}}{2}\right)$
3.  Refine the closest point by evaluating squared Euclidean distance for the 6 immediately adjacent $(\hat{q} \pm 1, \hat{r})$, $(\hat{q}, \hat{r} \pm1)$, $(\hat{q} \pm1, \hat{r}\pm1)$ lattice neighbors, selecting the point with minimal squared distance.
> This refinement step requires only 1-2 extra distance calculations, so total operations per snap are fixed regardless of input scale, satisfying O(1) latency. A deterministic tiebreaker rule is enforced for points exactly on Voronoi cell boundaries (e.g., preferring the point with loweraxial r-coordinate) to guarantee single-valued output.

## 3. Dodecet 12-Point Position Encoding

As defined:
A dodecet is a set of exactly 12 non-overlapping, spatially distinct lattice points surrounding a central $E_6$ anchor point, split into two concentric hexagonal shells:
1.  First nearest neighbors (6 points): Lattice points with squared Eisenstein norm =1, adjacent to the central anchor forming a regular hexagon.
2.  Second nearest neighbors (6 points): Lattice points with squared Eisenstein norm=3, forming a larger offset regular hexagon around the first shell.

## 4. Constraint Checking Mechanism
### 4.1 Full Rule Derivation
The compressed framing rule is fully decoded as follows:
1.  Compute snapped lattice points $L_a = \text{snap}(a) = (q_a, r_a)$ and $L_b = \text{snap}(b) = (q_b, r_b)$ for input spatial points $a,b$.
2.  Pack these axial coordinates into aligned fixed-width INT32 integers for batch processing, and compute the bitwise XOR difference $D = L_a \oplus L_b$.
3.  Calculate the Hamming weight of $D$ via hardware-accelerated popcount instructions.
4.  Flag the operation involving $a$ and $b$ as invalid if $\text{popcount}(D) > 2$.
> Threshold Rationale: A Hamming weight >2 corresponds to lattice points with a Manhattan axis separation greater than 2, or a Euclidean distance greater than ~2x the standard lattice side length, enforcing that only physically relevant close-proximity operations are validated.
