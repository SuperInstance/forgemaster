## Result 1: P2 Invariant Preservation Proof (COMPLETE)

### The Invariant
INV(σ) := ∃α: V → U such that ∀i, α(vi) ∈ Di and ∀c ∈ C, c(α(vj1),...,α(vjk)) holds.

In words: there exists a global assignment within current domains that satisfies all constraints.

### T1: REVISE(i,j,c) — INVARIANT PRESERVING ✓
**Proof:** If INV(σ) holds, there exists a solution α. For binary constraint c, α(vi) has support α(vj) in Dj. Since REVISE only removes values *without* support, α(vi) is not removed. Therefore α remains a valid solution for σ'. ∎

### T2: INTERSECT(i, mask) — NOT ALWAYS PRESERVING ✗
**Counterexample:** V={v1}, no constraints, D1={2}, mask={1}. INV holds initially (α(v1)=2). After intersection D1'=∅, INV fails.

**Exact preservation condition:** INV is preserved iff ∃ solution α of σ such that α(vi) ∈ mask.

### T3: UNION(i, mask) — ALWAYS PRESERVING ✓
**Proof:** Domains only grow. Any solution α for σ has α(vi) ∈ Di ⊆ Di', so α remains a solution. ∎

### T4: ASSERT(i, c) — HALT IMPLIES INV WAS FALSE ✓
**Proof by contradiction:** If INV(σ) holds, ∃ α with α(vi) ∈ Di satisfying c. Then ASSERT's check would pass (violation condition is false), so machine would not halt. Therefore if machine halts, INV(σ) was false. ∎

### AC-3 Termination Proof
**Measure:** μ(σ) = Σ|Di| (total domain sizes). Each REVISE either leaves μ unchanged or strictly decreases it. Since μ ≤ 64n (bounded) and μ ≥ 0, the natural number ordering is well-founded. Therefore AC-3 terminates.

### INV vs Arc Consistency
**Incomparable.** A CSP can be arc-consistent with no solution (3-cycle of ≠ on {1,2}). A CSP can have a solution without being arc-consistent (variable with unsupported value not used in any solution).

### Weakest Invariant
Without restrictions on T2 masks: only **True** (trivial). With restricted T2 (masks from constraint reasoning): INV itself is the weakest nontrivial invariant.

## Result 2: Minimum CSP Computational Primitives

### The 6-Instruction Machine
1. AND (domain intersection)
2. OR (domain union)
3. NOT (domain complement)
4. POPCOUNT (domain cardinality)
5. Branch on POPCOUNT (emptiness/cardinality test)
6. LOAD/STORE (access domain variables)

### Turing-Complete? YES
AND, OR, NOT form a functionally complete Boolean basis. Can construct XOR = (A∧¬B)∨(¬A∧B), then implement addition via full-adder logic, then arbitrary arithmetic. With conditional branch and unbounded memory, simulates a RAM machine.

### Solvable CSPs: ALL FINITE-DOMAIN
Any finite domain → bitmask. Constraints → Boolean relations on domain products. AC-3 + backtracking implementable. Machine solves all CSPs with finite bounded-size domains.

### Computational Class
The machine is a **concrete computational model for Boolean algebras**:
- AND = meet (∧), OR = join (∨), NOT = complement (¬)
- POPCOUNT = valuation (measure) giving cardinality
- Branch on POPCOUNT = test bottom element (emptiness)
- This implements **Stone's representation theorem** in hardware

### Transistor Count
Unlikely under 100 for practical word sizes:
- AND/OR: ~4 transistors each
- NOT: ~2 transistors
- POPCOUNT-64: >400 transistors (adder tree)
- For 4-bit word: might fit under 100, but domain limited to 4 values

### Thermodynamic Cost (Landauer's Principle)
- AND/OR: irreversible, ≥ n·kT·ln2 per n-bit operation
- NOT: reversible (bijection), ~zero dissipation in principle
- POPCOUNT: n → log₂(n+1) bits, dissipates (n - log₂(n+1))·kT·ln2 ≈ n·kT·ln2
- Branch: ~few bits of dissipation per branch

### Is This the Bottom?
**No.** Below Boolean algebra layer: physical logic gates (NAND/NOR), then transistors, then quantum mechanics, then Landauer limit. But for CSP solving specifically, these 6 primitives are the natural operations.

### Restriction to AND + NOT Only
Still Turing-complete (Sheffer stroke). But loses O(1) POPCOUNT — must compute cardinality serially in O(n). POPCOUNT is essential for efficient CSP solving.
